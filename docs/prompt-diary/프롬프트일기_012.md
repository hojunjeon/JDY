# [프롬프트 일기 #12] 백업 AGENTS 지침을 현재 AGENTS에 되살리기

> JDS를 만들면서 Codex와 어떻게 작업하는지 기록한다.
> 어떤 프롬프트가 들어왔고, 무엇이 바뀌었고, 그래서 어떤 결정이 남았는지 적는다.

---

## 그 시점의 게임

프로젝트 루트 `AGENTS.md`는 OMX가 생성한 오케스트레이션 규칙 중심으로 커져 있었다. 반면 `.AGENTS.md.bkup`에는 JDS 전용 협업 규칙, 사용자의 학습 맥락, 한국어 응답 선호, GitHub 저장소 기준 같은 프로젝트 맞춤 지침이 남아 있었다.

---

## 그날 상황

사용자는 백업 파일을 확인해 현재 `AGENTS.md`에 반영할 내용이 있는지 점검해 달라고 요청했다. 비교해 보니 현재 파일에는 일반적인 OMX 규칙은 충분했지만, 백업의 JDS 전용 응답 스타일과 작업 가드레일 일부가 빠져 있었다.

---

## 입력된 프롬프트

> ".AGENTS.md.bkup를 확인하고 AGENTS.md에 반영할 내용이 있는지 확인해줘"

---

## 무엇이 변화했나

`AGENTS.md`에 `JDS Project-Specific Guidance` 섹션을 추가했다. 이 섹션은 JDS 로컬 스킬 사용 기준, `DESIGN.md` 확인 조건, 선택적 서브에이전트 사용, 검증 증거 보고, 사용자 학습 수준에 맞춘 한국어 설명, 좁은 범위의 변경 원칙, 예상 GitHub 저장소를 한곳에 모은다.

백업의 `superpowers` 직접 언급은 현재 세션의 사용 가능한 스킬 표면과 맞지 않을 수 있어 그대로 복원하지 않고, “프로젝트 로컬 JDS 스킬을 적용한다”는 현재 환경에 맞는 표현으로 정리했다.

---

## 그래서 어떤 결정이 남았나

앞으로 JDS 작업에서는 일반 OMX 규칙 위에 JDS 전용 맥락을 함께 적용한다. 특히 사용자가 게임 개발을 통해 Codex와 개발 습관을 배우는 상황임을 고려해, 결과만 말하지 않고 필요한 경우 짧게 이유와 학습 포인트를 붙인다.

---

## 다음에 떠올릴 것

`omx setup`이나 AGENTS 재생성으로 루트 지침이 갱신되면, 이번에 되살린 `JDS Project-Specific Guidance`가 유지되는지 다시 확인해야 한다. 자동 생성 영역과 프로젝트 맞춤 영역을 분리해 두는 편이 다음 병합 비용을 줄인다.
